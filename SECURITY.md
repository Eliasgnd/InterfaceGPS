# Security Policy

## Supported versions

As this project is under active development, security fixes are applied to the
latest `main` branch.

## Reporting a vulnerability

Please do **not** open a public issue for suspected vulnerabilities.

Instead:

1. Open a private security advisory on GitHub (preferred), or
2. Contact the maintainer with a detailed report including:
   - affected component/module,
   - reproduction steps,
   - potential impact,
   - suggested mitigation (if known).

You can expect an acknowledgment within 7 days for valid reports.
